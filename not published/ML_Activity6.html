Discussion

Recall the three 3 main types of model selection algorithms:

  1. Variable selection
    Identify a subset of predictors that make the “best” predictive model of \(y\).
    Methods: best subset selection, backward stepwise selection, forward stepwise selection
    Drawbacks: gives overly optimistic conclusions about the significance of the remaining predictors.


  1. Shrinkage / regularization
    Fit a model with all predictors, but shrink / regularize their coefficients toward or to 0. This penalizes weakly informative predictors & creates sparse models.
    Methods: LASSO, ridge regression


  1. Dimension reduction
    Reduce the number of predictors by combining them into a smaller set of new predictors.
    Methods: principal components regression
    Drawbacks: predictors lose some meaning

Model selection: machine learning vs human learning

We’ve seen that model selection machine learning techniques help us focus the model building process. However, they usually leave us to make a final choice between multiple possible models. For example, recall the backward stepwise selection algorithm for the body fat percentage model:

We might logically argue for 4 different models:

  • 11 predictor model
    has the smallest CV MAE (identified by selectionFunction = "best")

  • 4 predictor model
    based on visual cues, this is where the CV MAE starts to plateau / stop improving

  • 2 predictor model
    the simplest model for which the CV MAE isn’t significantly different from the “best” 11 predictor model

  • 1 predictor model
    an average prediction error of ~3.85 body fat percentage points might not be practically different than the “best” average prediction error of ~3.65.

When it comes to picking a “final” model, there are some human considerations we must weigh:

  • goal
    How will this predictive model be used? Who will use it? Is it important that the model be easy for humans to interpret and apply (in which case we might favor simplicity)? Or is it just going to be utilized by a computer program (in which case we might favor the smallest prediction error)?

  • cost
    Data collection and storage require resources (eg: time, computer memory, and money). If these costs are high we might favor a simpler model.

LASSO: least absolute shrinkage and selection operator

Our goal is to build a regression model

\[y = \hat{\beta}_0 + \hat{\beta}_1 x_1 + \cdots + \hat{\beta}_k x_k + \varepsilon\]

We can summarize the residuals of each case \(i\) and the cumulative “residual sum of squares” (ie. sum of the squared residuals) by:

\[\begin{split} y_i - \hat{y}_i & = y_i - (\hat{\beta}_0 + \hat{\beta}_1x_{i1} + \cdots + \hat{\beta}_px_{ip}) \\ RSS & = \sum_{i=1}^n (y_i - \hat{y}_i)^2 \\ \end{split}\]

Criterion

Identify the model coefficients \(\hat{\beta}_1, \hat{\beta}_2, ... \hat{\beta}_p\) that minimize the penalized residual sum of squares:

\[RSS + \lambda \sum_{j=1}^p \vert \hat{\beta}_j\vert\]

Properties

  • As \(\lambda\) increases, coefficients shrink toward 0. This is called shrinkage or regularization.

  • Picking tuning parameter \(\lambda\) is a goldilocks problem, we donât want it to be too big or too small.
    • When \(\lambda = 0\) (too small), LASSO is equivalent to least squares, thus doesnât simplify the model.
    • When \(\lambda\) is too big it also kicks out useful predictors!

  • When \(\lambda\) is just right, LASSO helps build parsimonious or sparse models which only include the most relevant predictors. These are less prone to overfitting.

  • We can use cross-validation to help choose an appropriate \(\lambda\).

EXTRA: Bayesian Interpretation
IF you’ve taken Bayesian statistics, we can also write the LASSO as a Bayesian model. Specifically, LASSO estimates are equivalent to the posterior mode estimates of \(\beta_j\).

\[\begin{split} Y_i | \beta_0,...\beta_k & \sim N(\beta_0 + \beta_1X_1 + \cdots + \beta_k X_k, \sigma^2) \\ \beta_j & \sim \text{ Laplace (double-exponential)}(0, f(\lambda)) \\ \sigma^2 & \sim \text{ some prior} \\ \end{split}\]

caret: LASSO regression

To run the LASSO regression using caret, first load the package and set the seed:

library(caret)
set.seed(___)

You’ll then need to adapt the following code:

# Define a range of lambda tuning parameters to try
# The following is a starting point, but should be tweaked in each context
lambda_grid <- 10^seq(-3, 1, length = 100)

# Perform LASSO
lasso_model <- train(
    y ~ x,
    data = ___,
    method = "glmnet",
    tuneGrid = data.frame(alpha = 1, lambda = lambda_grid),
    trControl = trainControl(method = "cv", number = ___, selectionFunction = ___),
    metric = "MAE",
    na.action = na.omit
)
arguments meaning
y ~ . shorthand for the model of y by all possible predictors x in the data
data sample data which only includes y and potential predictors x
method glmnet implements the LASSO and other regularization methods
tuneGrid alpha = 1 specifies that we’re interested in the LASSO (not a different glmnet method). lambda = lambda_grid specifies to run the LASSO under each tuning parameter in lambda_grid.
trControl use cross-validation to train and test each tuning of the LASSO model. The process by which to pick a final model from among these is indicated by the selectionFunction argument, options including "best" and "oneSE". The best option will i.d. the model with the best metric. The oneSE option will i.d. the simplest model for which the metric is within one standard error of the best metric.
metric evaluate & compare competing LASSO models with respect to their MAE
na.action set na.action = na.omit to prevent errors if the data has missing values

# Identify which tuning parameter (ie. lambda) is "best"
