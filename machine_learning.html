Machine Learning - Intorduction

Concept of Machine Learning

Machine learning is an application of Artificial Intelligence (AI) that provides machines the ability to automatically learn and improve from data without being explicitly programmed. The main idea of Machine Learning is categorized into supervised learning and unsupervised learning.

Supervised learning is simply a process of learning algorithm from the training dataset. In this case, we would have input variables and an output variable and we use an algorithm to learn the function to map the output from the input. The goal of learning this function is that we can predict the output value when we have new input data. Within supervised learning, we have regression and classification.


Unsupervised learning is modeling the underlying or hidden structure or distribution in the data in order to learn more about them. Therefore, we would only have input data and no corresponding output variables. Within unsupervised learning, we have clustering and dimension reduction.

Example - Genre classification of songs on Spotify

[Note] This example contains classification and dimension reduction. I will also upload examples of regression and clustering soon.

This example is from a project I worked on in the introductry course of Machine Learning at Macalester: STAT 251 with Professor A. Johnson in 2019 Fall semester. In the example, I used a dataset of songs from Spotify to create an algorithm that predicts the genre of a song based on the song’s properties. Here is the link to the dataset: DOWNLOAD CSV file

For each song, we have information about the artist and the song’s properties such as names, age, duration, etc. I built an algorithm that automatically labels a song’s genre based on its properties and artist information. Thus, this is an example of classification, a supervised learning.

However, the dataset contains 57 observations and 19 features. Before creating the algorithm, we first need to reduce the number of features. This is an example of dimension reduction, an unsupervised learning. To do this, I used Principle Component Analysis (PCA). In short, PCA reduces the number of features by removing the correlation between the features, allowing the model to be more efficient and accurate.


The plot shows which variables had the most weights in constructing the first and second Principe Components (PCs). Energy, tempo, valence, acousticness were the significant portions of PC1. For PC2, the ages of the group members and daceability were significant. The plot tells us which variables are the key determinants of the song's genre.

After the transformation, I used the first 8 PCs to do classification, retaining about 80% of the information. Using the new data, I built a model using the KNN method to classify songs. KNN compare the songs and put the songs with similar feature values ("neighbors") in the same genre. Some examples of other methods we can use are classification tree and forest.

The overall accuracy of this model was 0.6604. Out of 53 songs without missing values in the dataset, 35 songs were classified accurately. The main limitation of the model is the prediction for ElectroClub songs. Only one of the 12 songs (8%) in this category was accurately classified. 23 of the 26 songs (88%) in the Pop category and 11 of 15 songs (73%) in the RBHiphop category were correctly classified.
